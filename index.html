人脑善于比较。事实上，没有比较，观念就不可能产生。正是通过比较这种精神活动，我们才能辨别事物的异同。

为了使论证正确有力，我们必须关注其事实（内容）和形式（结构）。

关注到真实性和有效性的区别是非常重要的。尽管人们常常将两者混淆，事实上它们差别很大。首先，真实性针对命题的内容，而有效性则针对命题的结构，我们称之为论证。其次，只要命题反映的事实是真相，它就是正确的。但一个论证如果想成立，如前文所述，在前提正确的基础上，它的结构也必须能有效支持它的结论。

现在我们来讨论三种简单的论证形式：联言论证、选言论证和条件论证。

选言论证就是说组成这个命题的两个部分是相互排斥的，它们不能同时为真。

三段论推理的根据是，首先确定某一部分是属于整体的，然后得出某一部分的组成成员也是属于整体的。

前面我们已经说过，一个三段论的论证，或者是任何一个论证，如果想要得到正确的结论，必须达到两个基本的要求：**一是它必须有正确的内容，二是它必须有合理的结构。**正确的内容取决于作为前提的命题真实与否。如果结构合理，但是内容不合理，会产生什么样的后果呢？让我们来看下面这个例子：

每只狗都有三个头。

所以，牧羊犬有三个头。

虽然前提的真实性是得出正确论证的必要条件，但也不是充分条件。如果前提不能有效地支持结论，它能起到的作用也是寥寥。回想一下，前提的作用是支持结论，给我们可以接受结论的理由。但是，如果前提本身和结论毫不相关，即便它是正确的，也不能起到应有的作用。前提失效的表现之一是不能证明结论。让我们来看下面的例子：

皮埃尔是个全美闻名的橄榄球运动员。

皮埃尔很英俊，笑容迷人。

所以，皮埃尔应该被选为州长。

音乐家都是杰出的人。

塞西莉亚是音乐家。

所以，塞西莉亚是杰出的人。

这两个的区别主要是第一个大前提是事实，而第二个则是断论

迭代器的类型

C++标准库（STL）中定义了几种不同类型的迭代器：

  1. 输入迭代器（Input Iterator）：只读访问，单向移动。
  2. 输出迭代器（Output Iterator）：只写访问，单向移动。
  3. 前向迭代器（Forward Iterator）：只读访问，单向移动，可以多次遍历。
  4. 双向迭代器（Bidirectional Iterator）：只读访问，可以双向移动。
  5. 随机访问迭代器（Random Access Iterator）：只读访问，可以双向移动，并且支持随机访问。
    #这里的迭代器很像智能指针，这一点为什么说智能指针要好于野指针呢？

智能指针（Smart Pointers）优于野指针（Dangling Pointers）有多个原因，主要在于智能指针通过自动化的内存管理和其他安全机制，帮助程序员避免常见的内存管理错误。以下是智能指针优于野指针的详细原因：

1. 自动内存管理

野指针：野指针是指向已经被释放的内存或者未初始化的指针。如果程序员不小心使用了野指针，会导致未定义行为，可能导致程序崩溃或其他严重错误。

智能指针：智能指针通过自动管理对象的生命周期来避免野指针的出现。例如，C++ 标准库提供的 std::unique_ptr 和 std::shared_ptr 可以在不再需要对象时自动释放内存。这样可以避免手动释放内存时的错误。

2. 资源泄漏防护

野指针：如果程序员忘记释放内存，会导致内存泄漏，进而造成程序占用过多的内存资源。

智能指针：智能指针通过 RAII（资源获取即初始化）原则管理资源，确保资源在不再使用时被自动释放。例如，当 std::unique_ptr 离开作用域时，它所管理的对象会被自动删除，避免了资源泄漏。

3. 线程安全

野指针：在多线程环境中，手动管理内存非常容易出错，多个线程可能会同时访问或修改同一块内存，导致数据竞争和未定义行为。

智能指针：std::shared_ptr 提供了线程安全的引用计数机制，多个线程可以安全地共享和管理同一个对象。这有助于避免在多线程环境中出现内存管理问题。

4. 简化代码和减少错误

野指针：手动内存管理增加了代码的复杂性，容易引入各种错误，如重复释放、未释放内存等。

智能指针：智能指针使代码更简洁、更安全。例如，使用 std::unique_ptr 和 std::shared_ptr 可以减少显式的 delete 操作，降低了错误的可能性。

#那么智能指针和迭代器的关系是什么样子的呢？

智能指针

智能指针是用于自动管理动态内存的指针，它们通过 RAII（资源获取即初始化）原则确保内存资源在不再需要时被自动释放。常见的智能指针包括 std::unique_ptr、std::shared_ptr 和 std::weak_ptr。

  • std::unique_ptr：独占所有权的智能指针，确保同一时间内只有一个指针可以指向对象。
  • std::shared_ptr：共享所有权的智能指针，多个指针可以共享同一个对象。
  • std::weak_ptr：与 std::shared_ptr 配合使用，不增加引用计数，避免循环引用导致的内存泄漏。

迭代器

迭代器是用于遍历容器（如数组、链表、集合等）中的元素的对象。迭代器提供了一种一致的方式来访问容器中的元素，支持多种操作符（如 *、++、--）来进行元素的访问和移动。常见的迭代器类型包括输入迭代器、输出迭代器、前向迭代器、双向迭代器和随机访问迭代器。

智能指针与迭代器的关系

虽然智能指针和迭代器在概念上是独立的，但在实践中，它们可以一起使用，特别是在以下场景中：

  1. 动态分配的容器元素：当容器的元素是动态分配的对象时，可以使用智能指针来管理这些对象的内存。

  2. 自定义容器的迭代器：如果你实现了一个自定义的容器，并且该容器的元素是通过智能指针来管理的，你可以在迭代器中使用智能指针来安全地访问和操作这些元素。

enum Rotation <: ToString & Hashable & Equatable {

F | B | L | R | U | D
| X(Rotation, Rotation) // 仓颉支持 enum 构造器和递归定义，此处 X 用于组织复合旋转操作
| I(Rotation) // I(r) 用于表示 r 的逆变换，即正对 r 面逆时针旋转 90 度

public operator func *(that: Rotation): Rotation {
    match (this) {
        case X(x1, x2) => X(x1, X(x2, that)) // 按此顺序分解重组，使得递归时按从左到右的顺序执行变换
        case _ => X(this, that)
    }

public operator func **(exp: UInt32): Rotation {
    var result = this
    for (_ in 0..(exp - 1)) {
        result = result * this
    }
    return result
}

// Permutation 可以表示一个置换序列
// 如 [(L, 2.c), (U, -0.r)] 表示 L 面第三列和 U 面第一行之间的轮换，且前者在换到后者时需要倒置
type Permutation = Array<(Face, Index)>
extend Permutation {
// 获取当前置换的逆变换
public func inverse() {
let perm = this.clone()
for (i in 0..perm.size) {
let j = (i + 1) % perm.size
let (face, index) = this[i] // 解构元组
if (let Inv(_) <- this[j][1]) { // if-let 模式匹配
perm[i] = (face, -(index.value()))
} else {
perm[i] = (face, index.value())
}
}
perm.reverse()
return perm
}

// 对指定面上的矩阵旋转 90 度，inverse = true 代表逆时针方向
// 此操作不影响其他面，要配合邻边置换操作才能完成魔方的实际转动
private func rotate(face: Face, inverse: Bool) {
    const N = 3
    let matrix = Matrix.create(face)
    let map = if (inverse) {
        { i: Int64, j: Int64 => (N - 1 - j, i) }
    } else {
        { i: Int64, j: Int64 => (j, N - 1 - i) }
    }
    for (i in 0..N) {
        for (j in 0..N) {
            let (u, v) = map(i, j)
            matrix[u][v] = data[face][i][j]
        }
    }
    data[face] = matrix
}

// 实现各面行/列之间的置换操作，输入是一个置换序列
private func permute(perm: Permutation, inverse: Bool) {
    let p = if (inverse) { perm.inverse() } else { perm.clone() }
    p.reverse()
    var (lastFace, lastIndex) = p[0]
    let vector = data[lastFace][lastIndex]
    for (i in 1..p.size) {
        let (face, index) = p[i]
        data[lastFace].set(lastIndex, data[face][index])
        (lastFace, lastIndex) = (face, index)
    }
    data[lastFace].set(lastIndex, vector)
}

    match (rotation) {
        case I(r) => transform(r, !inverse)
        case X(r1, r2) => // 对复合操作进行递归分解
            if (inverse) { // 逆变换需要反序
                transform(r2, inverse)
                transform(r1, inverse)
            } else {
                transform(r1, inverse)
                transform(r2, inverse)
            }
        case _ => // 各原子操作对应面先旋转 90 度，然后对相邻四个面执行置换操作
            rotate(rotation, inverse)
            permute(permutation[rotation], inverse)
    }
}
